\textbf{Introduction} This paper proposes a formal proposal to 
model the tonal processes resulting from vowel deletion. 
This paper proposes that tonal effects triggered by vowel 
deletion can be reduced to two operations over association 
relations: \textit{deassociation} and \textit{reassociation},
which is consistent with the core idea of autosegmental phonology
\citep{goldsmith1976} that prioeritize association lines modifcation
rtanther rhan segmentalfeatural change. We show that these two 
operations derive the major typological outcomes of vowel deletion, 
and extend to less common cases such as vowel deletion blocking. 
This provides a unified and computationally explicit account of 
tonal processes that preserves structural representations while 
modifying only association relations.

\textbf{Typology} \citet{kao2017phonological} conducted a 
typological survey of 30 languages that exhibit vowel deletion 
as a strategy for hiatus resolution, along with the resulting 
tonal processes. When the vowel is deleted, the original tone
that the deleted vowel is associated will undergo some re-arrangement
due to the unassociated relation. The common cases are: 1) get deleted;
2) get retented - either forma a contour with the adjacent tone or 
replace the neightbor tone. See Table 1.

\paragraph{Logical Transduction of Tonal Processes Triggered by Vowel
Deletion.}
Following \citet{strother2019diss,yolyan2025logical},
we represent a phonological representation as a relational structure
$M := \{D \mid R\}$, where $D$ is a finite domain and $R$ is a set of
relations.
More specifically, an AR of a tonal melody such as H.L can be represented
model-theoretically using a signature
consisting of unary and binary predicates
$R \in \{H, L, V, C, \lhd_s, \lhd_T, \alpha\}$,
which define the internal structure of ARs.
The binary predicates $\lhd_s(x,y)$ and $\lhd_T(x,y)$ refer to the
successor relations, that is, $x$ immediately follows $y$ on the
segmental tier and tonal tier, respectively.
The predicate $\alpha(x,y)$ encodes the association relation,
indicating that segment $x$ and tone $y$ are linked.These predicates, defined
in model-theoretic terms, express exactly the
same core ideas as autosegmental phonology \citep{goldsmith1976}, and reveal
greater simplicity as we delve deeper into tonal processes.

\input{ARmodels.tex}
Building on this view, we define a phonological transformation 
from an underlying representation (UR) to a surface 
representation (SR) is modeled as a relation between input
and output structures (see more in \citet{oakden2020notation}). 
The signature of the output can be obtained by referring to
the input signature. Formally, for any predicate $R(x)$ in the input signature,
a faithful output predicate $R'$ is defined as $R'(x) := R(x)$,
meaning that an element $x$ in the output domain has property $R$
iff it has property $R$ in the input. A binary relation follows the
same logic: $\lhd'(x,y) := \lhd(x,y)$
means that $x$ immediately follows $y$ in the output if and only if
it does so in the input.

\textbf{Our proposal} is that in vowel-deletion tonal processes,
all relations in the AR model remain faithful, i.e. $H'(x) := H(x), 
L'(x) := L(x)$ etc),
\textbf{except for the $\alpha$ relation}. Specifically:
\begin{enumerate}[noitemsep]
    \item The output association is the product of three operations:
    \textit{faithfulness}, \textit{deassociation}, and/or
\textit{reassociation}:
    \[
    \alpha'(x,y) := \bigl(\alpha(x,y) \land \neg \textsf{deassoc}(x,y)\bigr)
    \lor \textsf{reassoc}(x,y)
    \]
 \item Despite the name ``vowel deletion,'' no nodes or associations are
removed from the domainl, but only elements
    associated to at least one other element will be pronounced,
    licensed by $\phi_{\textsf{license}}(x)$:
    \[
    \phi_{\textsf{license}}(x) := \exists y\, \alpha(x,y) (or ???\lor
\alpha(y,x))
    \]
\item Deassociation and reassociation are language-specific, hinging
on the choice of deleting/surviving vowel in the hiatus. For instance,
in a language that deletes the first vowel and retains the second,
$\textsf{delete\_V}(x) := \exists y\, \textsf{hiatus}(y,x)$
and $\textsf{survive\_V}(x) := \exists y\, \textsf{hiatus}(x,y)$,
    \[
    \textsf{hiatus}(x,y):= V(x) \land V(y) \land \lhd_S(x,y)
    \]
\end{enumerate}
